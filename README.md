## 🧠 2. Stock Price Predictor — LSTM Deep Learning (POWERGRID.NS)

<div align="center">

[![Repo](https://img.shields.io/badge/📂%20GitHub-Stock--Price--Predictor-181717?style=for-the-badge&logo=github)](https://github.com/Anandsavran/Stock-Price-Predictor-LSTM)
![TensorFlow](https://img.shields.io/badge/TensorFlow-FF6F00?style=for-the-badge&logo=tensorflow&logoColor=white)
![Keras](https://img.shields.io/badge/Keras-D00000?style=for-the-badge&logo=keras&logoColor=white)

</div>

> A **4-layer stacked LSTM model** trained on 17 years of POWERGRID.NS market data (4,208 trading days) to forecast future closing prices with a final training loss of **0.0021**.

### 📊 Dataset

| Property | Detail |
|---|---|
| Stock | POWERGRID Corporation of India — `POWERGRID.NS` |
| Source | `yfinance` (NSE India live + historical) |
| Period | Oct 2007 – Nov 2024 |
| Total Records | **4,208 trading days** |
| Features | Close · Open · High · Low · Volume |
| Target | Closing Price (₹) |
| Price Range | ₹18.24 (min) → ₹348.71 (max) |

### 🏗️ Complete Pipeline

```
yfinance API  →  Raw OHLCV Data (4,208 rows)
      ↓
EDA & Visualization
  ├── Null check ✅ (0 missing values)
  ├── Closing/Opening/High/Volume trend charts (Matplotlib)
  ├── Candlestick chart (Plotly — interactive)
  ├── MA100 + MA200 (Simple Moving Average)
  └── EMA100 + EMA200 (Exponential Moving Average)
      ↓
Train / Test Split  →  70% (2,945 rows) | 30% (1,263 rows)
      ↓
MinMaxScaler  →  Normalize to [0, 1]
      ↓
Sliding Window — 100 days look-back
  x_train shape: (2845, 100, 1)
      ↓
4-Layer LSTM  →  50 Epochs  →  Adam  →  MSE Loss: 0.0021
      ↓
Inverse Scale → Predict in ₹  →  Actual vs Predicted Plot
      ↓
Model saved: stock_model.keras
```

### 🧬 Model Architecture

```python
model = Sequential([
    LSTM(50,  activation='relu', return_sequences=True, input_shape=(100,1)),
    Dropout(0.2),
    LSTM(60,  activation='relu', return_sequences=True),
    Dropout(0.3),
    LSTM(80,  activation='relu', return_sequences=True),
    Dropout(0.4),
    LSTM(120, activation='relu'),
    Dropout(0.5),
    Dense(1)
])
# Total Parameters: 178,761
# Optimizer: Adam | Loss: MSE | Epochs: 50 | Final Loss: 0.0021
```

## 🧬 3. NeuroLoom — Brainwave-to-Image AI System *(Patent Filed)*

<div align="center">

> **Patent-filed AI system** that reads real-time EEG brainwave signals via a headset and converts them into dynamic, continuously evolving digital artwork using GANs and Neural Style Transfer — bridging neuroscience, AI, and creative art.

### 💡 What It Does

```
You wear an EEG headset
         ↓
NeuroLoom reads your brain activity in real-time
         ↓
AI classifies your mental state (Relaxed / Focused / Creative / Excited)
         ↓
GAN generates evolving digital art based on your mental state
         ↓
Art displayed on screen / VR / AR / LED wall — updating live
```

### 🧠 Brainwave → Art Mapping

| Brainwave | Frequency | Mental State | Color Palette | Stroke Style | Speed |
|---|---|---|---|---|---|
| **Delta** | 0.5–4 Hz | Deep sleep | Dark purples | Slow, heavy | Very slow |
| **Theta** | 4–8 Hz | Meditation / Creative | Indigo, violet | Dreamy, abstract | Medium |
| **Alpha** | 8–14 Hz | Relaxed / Calm | Soft blues, greens | Fluid, smooth | Slow |
| **Beta** | 14–30 Hz | Focused / Alert | Reds, yellows | Sharp, angular | Fast |
| **Gamma** | 30–100 Hz | High excitement | Bright white, orange | Chaotic, explosive | Very fast |

### 🏗️ System Pipeline

```
EEG Headset (OpenBCI / Muse / Emotiv)
          ↓
Signal Preprocessing
  ├── Noise reduction & artifact removal
  ├── FFT / Wavelet Transform / ICA filtering
  └── Power Spectral Density (PSD) analysis
          ↓
Cognitive Classification (ML)
  ├── SVM / k-NN / Deep Neural Networks
  └── Output: mental state label
          ↓
AI Art Generation Engine
  ├── GANs — generate novel conditioned artwork
  ├── Neural Style Transfer (NST) — apply artistic styles
  └── Visual parameter mapping (color, stroke, speed)
          ↓
Display Output
  └── Screen / VR Headset / AR Overlay / LED Wall / Projection
```

## 🔒 6. Deadlock Detection — Resource Allocation Graph Simulator

<div align="center">

[![Repo](https://img.shields.io/badge/📂%20GitHub-Deadlock--Detection--RAG-181717?style=for-the-badge&logo=github)](https://github.com/Anandsavran/Deadlock-Detection-RAG)
![Python](https://img.shields.io/badge/Python-3776AB?style=for-the-badge&logo=python&logoColor=white)
![NetworkX](https://img.shields.io/badge/NetworkX-Graph%20Library-orange?style=for-the-badge)

</div>

> Interactive Python GUI simulator that visualizes **Resource Allocation Graphs (RAG)** in real-time and detects deadlocks using NetworkX cycle detection — built with Tkinter + Matplotlib embedded live.

### 💡 What Is a Deadlock?

```
In OS: Process A holds R1 and waits for R2
       Process B holds R2 and waits for R1
       → Neither can proceed → DEADLOCK

Visualized as a cycle in the RAG:

P1 ──requests──► R1 ──allocated──► P2
▲                                    │
└──────── requests ◄── R2 ◄─────── ┘
       CYCLE DETECTED → 🔴 DEADLOCK!
```

### 🏗️ 3-Module Architecture

```
┌───────────────────────────────────────────────────────┐
│                  main.py (Entry Point)                │
└────────────────┬──────────────────┬───────────────────┘
                 │                  │
        ┌────────▼───────┐  ┌───────▼────────────┐
        │   Module 1     │  │     Module 2        │
        │     GUI        │  │   GraphHandler      │
        │  (Tkinter)     │  │  (NetworkX Logic)   │
        └────────┬───────┘  └───────┬─────────────┘
                 │                  │
                 └────────┬─────────┘
                          │
                 ┌────────▼───────────┐
                 │     Module 3       │
                 │  GraphVisualizer   │
                 │ (Matplotlib/plt)   │
                 └────────────────────┘
```

| Module | Class | Key Method | Responsibility |
|---|---|---|---|
| 1 | `GUI` | Button callbacks | Tkinter UI — 6 action buttons |
| 2 | `GraphHandler` | `nx.simple_cycles()` | DiGraph management + deadlock detection |
| 3 | `GraphVisualizer` | `FigureCanvasTkAgg` | Embedded live graph rendering |

### 🎨 Graph Colour Coding

| Element | Colour | Meaning |
|---|---|---|
| Process nodes | 🔵 Light Blue | Running processes (P1, P2, P3…) |
| Resource nodes | ⬜ Gray | System resources (R1, R2…) |
| Request edge | 🟠 Orange | Process waiting for resource |
| Allocation edge | 🟢 Green | Resource assigned to process |
| Deadlock cycle | 🔴 Red (planned) | Cycle nodes highlighted |

### 🔑 Core Code — Deadlock Detection

```python
def detect_deadlock(self):
    cycles = list(nx.simple_cycles(self.graph))
    if cycles:
        messagebox.showwarning("Deadlock Detected!", f"Cycle found: {cycles}")
    else:
        messagebox.showinfo("Safe State", "No deadlock detected ✅")
```

### 🎮 Step-by-Step Usage

```
1. Click "Add Process" → Enter P1, P2, P3
2. Click "Add Resource" → Enter R1, R2
3. Click "Request Resource" → P1 requests R1 (orange arrow)
4. Click "Allocate Resource" → R1 allocated to P2 (green arrow)
5. Click "Request Resource" → P2 requests R2 (orange arrow)
6. Click "Allocate Resource" → R2 allocated to P1 (green arrow)
7. Click "Detect Deadlock"
   → ⚠️ WARNING: Deadlock in [['P1', 'R1', 'P2', 'R2']]
```

### 🔔 Daily Tips Notification Flow

```
User toggles Daily Tips Switch ON
          ↓
showTestNotification()
→ Immediate notification appears in system tray
→ Confirms permission is working
          ↓
TipsHelper.enableDailyTip(context, hour=9, minute=0)
→ Persists enabled=true + time to SharedPreferences
→ Schedules AlarmManager.setInexactRepeating alarm
          ↓
          ⏰ Every day at 09:00...
          ↓
TipReceiver (BroadcastReceiver) fires
→ Random tip selected from pool
→ NotificationHelper.showNotification() → system tray
→ TipsRepository.addTip(text, timestamp) → JSON array in SharedPreferences
          ↓
TipsActivity → RecyclerView → shows all tips (latest first)
          ↓
Device Reboots?
→ BootReceiver fires on RECEIVE_BOOT_COMPLETED
→ Re-schedules alarm automatically ✅
```
